## Goal

Create and apply all PostgreSQL database migrations for the R2P platform. Every downstream service depends on these tables. Migrations must be idempotent, reversible, and run automatically on startup.

---

## Tables

| Table | Purpose |
|-------|---------|
| `R2PRequest` | Core request entity — owns the state machine |
| `R2PStateTransition` | Append-only log of every state change |
| `R2PAcknowledgement` | Timestamped delivery acknowledgements |
| `R2PResponse` | Payer responses (accept / decline / defer) |
| `R2PPayment` | Payment records linked to accepted R2P |
| `EventSubscription` | Participant webhook registrations |
| `AuditStore` | Append-only audit log — all events |
| `OutboxEvent` | Outbound webhook delivery queue |

## Key Constraints

### R2PRequest — optimistic locking
```sql
version INTEGER NOT NULL DEFAULT 0
```
Every write must check `WHERE id = $1 AND version = $expected` and increment `version`. A mismatch returns 409.

### AuditStore — insert-only
A DB-level trigger blocks `UPDATE` and `DELETE` on `AuditStore`. No application code path may mutate an existing audit row.

```sql
CREATE OR REPLACE FUNCTION audit_store_immutable()
RETURNS trigger LANGUAGE plpgsql AS $$
BEGIN
  RAISE EXCEPTION 'AuditStore rows are immutable';
END;
$$;

CREATE TRIGGER audit_store_no_update
  BEFORE UPDATE OR DELETE ON "AuditStore"
  FOR EACH ROW EXECUTE FUNCTION audit_store_immutable();
```

### OutboxEvent — delivery lifecycle
```
status: 'pending' | 'delivered' | 'failed'
retry_count: INTEGER NOT NULL DEFAULT 0
next_retry_at: TIMESTAMPTZ (nullable — null means no retry scheduled)
```

## Migration Contract

Each `.sql` file has an `-- up` block and a `-- down` block.

```sql
-- up
CREATE TABLE IF NOT EXISTS "R2PRequest" (
  id            UUID PRIMARY KEY,
  idempotency_key TEXT NOT NULL UNIQUE,
  payer_id      TEXT NOT NULL,
  payee_id      TEXT NOT NULL,
  originating_participant_id TEXT NOT NULL,
  receiving_participant_id   TEXT NOT NULL,
  amount        NUMERIC(18,2) NOT NULL,
  currency      CHAR(3) NOT NULL,
  due_date      DATE NOT NULL,
  expiry_timestamp TIMESTAMPTZ NOT NULL,
  remittance_info  TEXT,
  status        TEXT NOT NULL DEFAULT 'created',
  version       INTEGER NOT NULL DEFAULT 0,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- down
DROP TABLE IF EXISTS "R2PRequest";
```
