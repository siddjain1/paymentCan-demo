## Goal

Stateless middleware that validates all inbound and outbound R2P messages against ISO 20022 field definitions. Versioned independently of business logic — schema updates deploy without changing validator code. Every downstream service calls this before processing a message.

---

## Message Types

| Message Type | Purpose | Direction |
|-------------|---------|-----------|
| `pain.013` | Request-to-Pay initiation | Inbound (from originating participant) |
| `pain.014` | Payer response (accept/decline/defer) | Inbound (from receiving participant) |
| `camt.087` | Modification request | Inbound (from originating participant) |

## Core Types

```typescript
// src/validators/iso20022.ts

export type MessageType = 'pain.013' | 'pain.014' | 'camt.087'

export interface FieldError {
  field: string
  message: string
}

export type ValidationResult =
  | { valid: true }
  | { valid: false; code: 'VALIDATION_ERROR'; fields: FieldError[] }

export function validate(
  messageType: MessageType,
  payload: Record<string, unknown>
): ValidationResult
```

---

## Schema Definition Format

Each schema JSON file defines required fields and their types. Validator loads file based on `messageType` + `SCHEMA_VERSION` env var.

```json
// pain.013.2019.json
{
  "messageType": "pain.013",
  "version": "2019",
  "required": [
    { "field": "payerId",          "type": "string" },
    { "field": "payeeId",          "type": "string" },
    { "field": "amount",           "type": "number" },
    { "field": "currency",         "type": "string", "length": 3 },
    { "field": "dueDate",          "type": "isoDate" },
    { "field": "expiryTimestamp",  "type": "isoDatetime" },
    { "field": "idempotencyKey",   "type": "string" }
  ],
  "optional": [
    { "field": "remittanceInfo",   "type": "string" }
  ]
}
```

Supported type tokens: `string`, `number`, `boolean`, `isoDate` (YYYY-MM-DD), `isoDatetime` (ISO 8601 with timezone).

---

## Versioning

Schema version is resolved at startup from `SCHEMA_VERSION` env var (default: `'2019'`). The validator loads schema files with `{messageType}.{version}.json` naming. No code changes required to add a new schema version — drop in a new JSON file and update the env var.

```typescript
const SCHEMA_VERSION = process.env.SCHEMA_VERSION ?? '2019'

function loadSchema(messageType: MessageType): SchemaDefinition {
  // loads src/validators/schemas/{messageType}.{SCHEMA_VERSION}.json
}
```

## Express Middleware Wrapper

```typescript
// src/validators/middleware.ts

export function validateISO20022(messageType: MessageType): RequestHandler {
  return (req, res, next) => {
    const result = validate(messageType, req.body)
    if (!result.valid) {
      return res.status(400).json({
        code: result.code,
        fields: result.fields,
      })
    }
    next()
  }
}
```

Usage in route:
```typescript
router.post('/r2p/requests', validateISO20022('pain.013'), createRequest)
```

## pain.013 Required Fields

| Field | Type | Constraint |
|-------|------|-----------|
| `payerId` | string | non-empty |
| `payeeId` | string | non-empty |
| `amount` | number | > 0 |
| `currency` | string | exactly 3 chars (ISO 4217) |
| `dueDate` | isoDate | YYYY-MM-DD |
| `expiryTimestamp` | isoDatetime | ISO 8601 with timezone, must be future |
| `idempotencyKey` | string | non-empty |
| `remittanceInfo` | string | optional |

## pain.014 Required Fields

| Field | Type | Constraint |
|-------|------|-----------|
| `r2pId` | string | non-empty UUID |
| `responseType` | string | one of: accept, decline, defer |
| `participantId` | string | non-empty |
| `respondedAt` | isoDatetime | ISO 8601 with timezone |

## camt.087 Required Fields

| Field | Type | Constraint |
|-------|------|-----------|
| `r2pId` | string | non-empty UUID |
| `participantId` | string | non-empty |
| `modifications` | object | at least one key of: amount, dueDate, expiryTimestamp, remittanceInfo |

## Acceptance Criteria

- [ ] `validate('pain.013', payload)` returns `{ valid: true }` for a fully valid payload
- [ ] Missing required field returns `{ valid: false, code: 'VALIDATION_ERROR', fields: [{ field, message }] }`
- [ ] Wrong type (e.g. amount as string) returns field-level error
- [ ] Invalid isoDate / isoDatetime format returns field-level error
- [ ] Valid messages pass through with no modification to payload
- [ ] `SCHEMA_VERSION` env var controls which schema files are loaded
- [ ] Unknown `messageType` throws a descriptive error (not silently passes)
- [ ] `validateISO20022(messageType)` middleware returns 400 with JSON error body on invalid input
- [ ] All three message types (pain.013, pain.014, camt.087) have schema files and tests
- [ ] All unit tests pass

---

## Out of Scope

- Full XSD validation (POC uses field-presence + type checks; XSD is production concern)
- Outbound message validation (inbound only for POC)
- Business rule validation (e.g. expiry must be after dueDate) — belongs in R2P Request Service
