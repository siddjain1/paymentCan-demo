## Goal

In-memory static map that resolves a payer proxy identifier (email, phone, or alias) to a routable participant endpoint and account reference. Used by the R2P Request Service (ticket 2.1) during request creation. No database calls — pure in-memory lookup backed by the seed data from ticket 1.1.

## Core Types

```typescript
// src/services/addressDirectory.ts

export type ProxyType = 'email' | 'phone' | 'alias'

export interface ParticipantAddress {
  participantId: string
  participantEndpoint: string
  accountRef: string
  ttlSeconds: number
}

export function resolve(
  proxyType: ProxyType,
  proxyValue: string
): ParticipantAddress | null
```

---

## In-Memory Map

Re-use `PARTICIPANT_MAP` from `src/db/seed.ts`. The Address Directory does NOT re-define participant data — it imports and wraps the existing map with a typed lookup interface.

```typescript
import { PARTICIPANT_MAP } from '../db/seed'

// PARTICIPANT_MAP key format: '{proxyType}:{proxyValue}'
// e.g. 'email:payer@banka.ca' → ParticipantAddress
```

Lookup:
```typescript
export function resolve(proxyType: ProxyType, proxyValue: string): ParticipantAddress | null {
  const key = `${proxyType}:${proxyValue}`
  return PARTICIPANT_MAP.get(key) ?? null
}
```

---

## Express Router

```
GET /internal/address-directory/resolve?proxyType={type}&proxyValue={value}
```

### Success (200)
```json
{
  "participantId": "BANK_A",
  "participantEndpoint": "http://localhost:4001",
  "accountRef": "ACC-BANKA-001",
  "ttlSeconds": 300
}
```

### 400 — Invalid proxy type
```json
{ "code": "INVALID_PROXY_TYPE", "message": "proxyType must be one of: email, phone, alias" }
```

### 404 — Not found
```json
{ "code": "PARTICIPANT_NOT_FOUND", "message": "No participant found for given proxy" }
```

## Acceptance Criteria

- [ ] `resolve('email', 'payer@banka.ca')` returns correct `ParticipantAddress` for BANK_A
- [ ] `resolve('phone', '+16135550001')` returns correct entry for BANK_C
- [ ] `resolve('alias', 'corp-alias-1')` returns correct entry for BANK_E
- [ ] `resolve(anyType, 'unknown@value.ca')` returns `null`
- [ ] `GET /internal/address-directory/resolve?proxyType=email&proxyValue=payer@banka.ca` returns 200 with correct JSON
- [ ] Unknown `proxyValue` returns 404 `PARTICIPANT_NOT_FOUND`
- [ ] Invalid `proxyType` (e.g. `fax`) returns 400 `INVALID_PROXY_TYPE`
- [ ] Missing `proxyType` or `proxyValue` query param returns 400
- [ ] All unit tests pass

## Out of Scope

- Database-backed participant registry (production concern)
- TTL cache invalidation (POC uses static TTL of 300s)
- Participant self-registration API
- Multi-proxy per participant (each participant has exactly one proxy in POC)
