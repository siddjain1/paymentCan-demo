## Goal

Core R2P request creation endpoint. Validates the inbound pain.013 message, checks idempotency, resolves the payer's proxy to a participant, generates a globally unique transaction ID, persists the request, and records the initial audit trail.

## API Contract

```
POST /r2p/requests
  Middleware: validateISO20022('pain.013')
  Request:  {
    payerId:          string   // proxy value for payer lookup
    payeeId:          string
    amount:           number
    currency:         string   // ISO 4217, 3 chars
    dueDate:          string   // YYYY-MM-DD
    expiryTimestamp:  string   // ISO 8601 with timezone
    remittanceInfo?:  string
    idempotencyKey:   string
  }
  Response 201: { r2pId: string, status: "created", createdAt: string }
  Errors:
    400 VALIDATION_ERROR        — pain.013 schema invalid (caught by middleware)
    404 PAYER_NOT_FOUND         — payer proxy not in Address Directory
    409 DUPLICATE_REQUEST       — idempotencyKey already used
```

## Service: createRequest()

```typescript
// src/services/r2pRequest.ts

export interface CreateRequestInput {
  payerId: string
  payeeId: string
  amount: number
  currency: string
  dueDate: string
  expiryTimestamp: string
  remittanceInfo?: string
  idempotencyKey: string
}

export type CreateRequestResult =
  | { ok: true;  r2pId: string; status: 'created'; createdAt: string }
  | { ok: false; code: 'DUPLICATE_REQUEST' | 'PAYER_NOT_FOUND'; message: string }

export function createRequest(input: CreateRequestInput): CreateRequestResult
```

### Execution steps

1. **Idempotency check** — `r2pRepo.findByIdempotencyKey(input.idempotencyKey)`
   - If found: return `{ ok: false, code: 'DUPLICATE_REQUEST' }`

2. **Resolve payer proxy** — `resolve('email' | 'phone' | 'alias', input.payerId)`
   - Attempt `email` first, then `phone`, then `alias` — or require `payerProxyType` field
   - Simpler for POC: treat `payerId` as email, try resolve; if null → PAYER_NOT_FOUND
   - If null: return `{ ok: false, code: 'PAYER_NOT_FOUND' }`

3. **Generate r2pId** — UUID v7 (time-ordered):
   ```typescript
   function generateR2PId(): string {
     // Prefix with timestamp ms in hex for ordering, suffix with random
     const ts = Date.now().toString(16).padStart(12, '0')
     const rand = crypto.randomUUID().replace(/-/g, '').slice(12)
     return `${ts.slice(0,8)}-${ts.slice(8)}-7${rand.slice(0,3)}-${rand.slice(3,7)}-${rand.slice(7,19)}`
   }
   ```

4. **Persist request** — `r2pRepo.save(row)` where:
   ```typescript
   const row: R2PRequestRow = {
     id: r2pId,
     idempotency_key: input.idempotencyKey,
     payer_id: input.payerId,
     payee_id: input.payeeId,
     originating_participant_id: 'PLATFORM',   // POC: platform is originator
     receiving_participant_id: resolved.participantId,
     amount: input.amount,
     currency: input.currency,
     due_date: input.dueDate,
     expiry_timestamp: input.expiryTimestamp,
     remittance_info: input.remittanceInfo ?? null,
     status: 'created',
     version: 0,
     created_at: new Date().toISOString(),
     updated_at: new Date().toISOString(),
   }
   ```

5. **Audit trail** — `auditRepo.append({ r2p_id: r2pId, event_type: 'REQUEST_CREATED', actor: 'system', detail: JSON.stringify(input) })`

6. **State transition** — `transitionRepo.append({ r2p_id: r2pId, from_status: null, to_status: 'created', actor: 'system' })`

7. **Return** — `{ ok: true, r2pId, status: 'created', createdAt: row.created_at }`

## Route Handler

```typescript
// src/routes/r2pRequests.ts
router.post(
  '/r2p/requests',
  validateISO20022('pain.013'),   // 400 on schema failure
  (req, res) => {
    const result = createRequest(req.body)
    if (!result.ok) {
      const status = result.code === 'DUPLICATE_REQUEST' ? 409 : 404
      return res.status(status).json({ code: result.code, message: result.message })
    }
    return res.status(201).json({ r2pId: result.r2pId, status: result.status, createdAt: result.createdAt })
  }
)
```

## Acceptance Criteria

- [ ] Valid request returns 201 `{ r2pId, status: "created", createdAt }`
- [ ] `r2pId` is a UUID v7 (starts with time-ordered hex prefix)
- [ ] Duplicate `idempotencyKey` returns 409 DUPLICATE_REQUEST
- [ ] Invalid pain.013 payload returns 400 VALIDATION_ERROR with field detail
- [ ] Unknown payer proxy returns 404 PAYER_NOT_FOUND
- [ ] Request persisted to in-memory store with all fields correct
- [ ] Audit entry appended with event_type `REQUEST_CREATED`
- [ ] State transition appended: `null → created`
- [ ] `resetStore()` between tests ensures isolation
- [ ] All unit tests pass
