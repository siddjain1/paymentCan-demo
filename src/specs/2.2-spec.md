# Spec: 2.2 — PATCH /r2p/requests/{id} — Modify Request

**Ticket:** 2.2
**Epic:** 2 — Payee Journey: Request Initiation
**Type:** Backend API
**Points:** 3
**Depends on:** 2.1 ✓
**Status:** Approved

---

## Goal

Allow the originating participant to modify permissible fields on an R2P request before the payer accepts it. Enforces state guard, validates modified values, applies optimistic locking, and records all changes in the audit trail.

---

## API Contract

```
PATCH /r2p/requests/:r2pId
  Request:  {
    amount?:           number   // > 0
    dueDate?:          string   // YYYY-MM-DD
    expiryTimestamp?:  string   // ISO 8601 with timezone
    remittanceInfo?:   string
  }
  Response 200: { r2pId: string, status: string, updatedAt: string }
  Errors:
    400 VALIDATION_ERROR        — provided field values are invalid
    400 NO_FIELDS_TO_UPDATE     — patch body is empty or has no recognised fields
    404 NOT_FOUND               — r2pId does not exist
    409 INVALID_STATE_TRANSITION — request is not in created or sent state
```

---

## Service: modifyRequest()

Add to `src/services/r2pRequest.ts`:

```typescript
export interface ModifyRequestInput {
  amount?: number
  dueDate?: string
  expiryTimestamp?: string
  remittanceInfo?: string
}

export type ModifyRequestResult =
  | { ok: true;  r2pId: string; status: string; updatedAt: string }
  | { ok: false; code: 'NOT_FOUND' | 'INVALID_STATE_TRANSITION' | 'NO_FIELDS_TO_UPDATE' | 'VALIDATION_ERROR'; message: string }

export function modifyRequest(r2pId: string, patch: ModifyRequestInput): ModifyRequestResult
```

### Execution steps

1. **Empty patch check** — if no recognised keys in patch → `{ ok: false, code: 'NO_FIELDS_TO_UPDATE' }`

2. **Fetch** — `r2pRepo.findById(r2pId)` → `{ ok: false, code: 'NOT_FOUND' }` if null

3. **State guard** — current status must be `created` or `sent`:
   ```typescript
   const MODIFIABLE_STATES = ['created', 'sent']
   if (!MODIFIABLE_STATES.includes(current.status)) {
     return { ok: false, code: 'INVALID_STATE_TRANSITION',
              message: `Cannot modify request in state: ${current.status}` }
   }
   ```

4. **Field validation** — validate only the fields present in patch:
   - `amount`: must be `number`, `> 0`, finite
   - `dueDate`: must match `/^\d{4}-\d{2}-\d{2}$/` and be a valid date
   - `expiryTimestamp`: must parse as valid ISO 8601 with timezone
   - `remittanceInfo`: must be `string` (empty string allowed)
   - Any invalid field → `{ ok: false, code: 'VALIDATION_ERROR', message: '...' }`

5. **Apply patch** — `r2pRepo.update(r2pId, { ...patch, updated_at: now }, current.version)`
   - Optimistic lock is enforced inside `r2pRepo.update()` — will throw if version mismatch

6. **Audit** — `auditRepo.append({ r2p_id, event_type: 'REQUEST_MODIFIED', actor: 'system', detail: JSON.stringify(patch) })`

7. **Transition record** — `transitionRepo.append({ r2p_id, from_status: current.status, to_status: current.status, actor: 'system', reason: 'modification' })`

8. **Return** — `{ ok: true, r2pId, status: updated.status, updatedAt: updated.updated_at }`

## Route

Add to `src/routes/r2pRequests.ts`:

```typescript
router.patch('/r2p/requests/:r2pId', (req, res) => {
  const result = modifyRequest(req.params.r2pId, req.body)
  if (!result.ok) {
    const statusMap: Record<string, number> = {
      NOT_FOUND: 404,
      INVALID_STATE_TRANSITION: 409,
      NO_FIELDS_TO_UPDATE: 400,
      VALIDATION_ERROR: 400,
    }
    return res.status(statusMap[result.code] ?? 400).json({ code: result.code, message: result.message })
  }
  return res.status(200).json({ r2pId: result.r2pId, status: result.status, updatedAt: result.updatedAt })
})
```

## Acceptance Criteria

- [ ] Valid patch returns 200 `{ r2pId, status, updatedAt }`
- [ ] Patched fields are persisted correctly in the store
- [ ] Non-existent `r2pId` returns 404 NOT_FOUND
- [ ] Request in `delivered`, `accepted`, `paid`, or any terminal state returns 409 INVALID_STATE_TRANSITION
- [ ] Empty patch body returns 400 NO_FIELDS_TO_UPDATE
- [ ] Invalid field value (e.g. amount = -1, bad date) returns 400 VALIDATION_ERROR
- [ ] Modification recorded in audit store with event_type `REQUEST_MODIFIED`
- [ ] Transition record appended (same status → same status)
- [ ] `resetStore()` between tests ensures isolation
- [ ] All unit tests pass

---

## Out of Scope

- Notifying the receiving participant of the modification (ticket 6.5 — Event Publisher)
- Modifying `payerId`, `payeeId`, `currency` (immutable fields — business rule)
- Bulk modification of multiple requests
