# Spec: 2.3 — DELETE /r2p/requests/{id} — Cancel Request

**Ticket:** 2.3
**Epic:** 2 — Payee Journey: Request Initiation
**Type:** Backend API
**Points:** 3
**Depends on:** 2.1 ✓, 2.2 ✓
**Status:** Approved

---

## Goal

Allow the originating participant to cancel an active R2P request before the payer accepts it. Enforces a state guard limiting cancellation to pre-acceptance states, persists the status change with optimistic locking, and records the cancellation in the audit trail.

---

## API Contract

```
DELETE /r2p/requests/:r2pId
  Response 200: { r2pId: string, status: "cancelled", cancelledAt: string }
  Errors:
    404 NOT_FOUND                  — r2pId does not exist
    409 INVALID_STATE_TRANSITION   — request is in a post-acceptance state
```

---

## Service: cancelRequest()

Add to `src/services/r2pRequest.ts`:

```typescript
export type CancelRequestResult =
  | { ok: true;  r2pId: string; status: 'cancelled'; cancelledAt: string }
  | { ok: false; code: 'NOT_FOUND' | 'INVALID_STATE_TRANSITION'; message: string }

export function cancelRequest(r2pId: string): CancelRequestResult
```

### Execution steps

1. **Fetch** — `r2pRepo.findById(r2pId)` → `{ ok: false, code: 'NOT_FOUND' }` if null

2. **State guard** — current status must be `created`, `sent`, or `delivered`:
   ```typescript
   const CANCELLABLE_STATES = ['created', 'sent', 'delivered']
   if (!CANCELLABLE_STATES.includes(current.status)) {
     return { ok: false, code: 'INVALID_STATE_TRANSITION',
              message: `Cannot cancel request in state: ${current.status}` }
   }
   ```

3. **Apply cancellation** — `r2pRepo.update(r2pId, { status: 'cancelled', updated_at: now }, current.version)`

4. **Audit** — `auditRepo.append({ r2p_id, event_type: 'REQUEST_CANCELLED', actor: 'system', detail: JSON.stringify({ previousStatus: current.status }) })`

5. **State transition** — `transitionRepo.append({ r2p_id, from_status: current.status, to_status: 'cancelled', actor: 'system' })`

6. **Return** — `{ ok: true, r2pId, status: 'cancelled', cancelledAt: now }`

---

## Route

Add to `src/routes/r2pRequests.ts`:

```typescript
router.delete('/r2p/requests/:r2pId', (req, res) => {
  const result = cancelRequest(req.params.r2pId)
  if (!result.ok) {
    const statusMap: Record<string, number> = {
      NOT_FOUND: 404,
      INVALID_STATE_TRANSITION: 409,
    }
    return res.status(statusMap[result.code] ?? 400).json({ code: result.code, message: result.message })
  }
  return res.status(200).json({ r2pId: result.r2pId, status: result.status, cancelledAt: result.cancelledAt })
})
```

## Acceptance Criteria

- [ ] Valid cancel returns 200 `{ r2pId, status: "cancelled", cancelledAt }`
- [ ] Requests in `created`, `sent`, and `delivered` states are cancellable
- [ ] Requests in `accepted`, `payment_processing`, `paid`, `payment_failed`, `expired`, or `cancelled` return 409 INVALID_STATE_TRANSITION
- [ ] Non-existent `r2pId` returns 404 NOT_FOUND
- [ ] Cancellation recorded in audit store with event_type `REQUEST_CANCELLED`
- [ ] State transition recorded: `{previousStatus} → cancelled`
- [ ] `resetStore()` between tests ensures isolation
- [ ] All unit tests pass
