## Goal

Deliver a created R2P request to the receiving participant's endpoint via HTTPS POST. Transitions status from `created` to `sent` on dispatch, records delivery state in AuditStore, and marks delivery `confirmed` on HTTP 2xx. Not a REST endpoint — internal trigger only.

## Service: dispatch()

```typescript
// src/services/routingEngine.ts

export interface DeliveryResult {
  status: 'delivered' | 'failed'
  statusCode?: number
}

export type HttpClient = (url: string, body: unknown) => Promise<{ statusCode: number }>

// Injectable — swap in tests via setHttpClient()
let httpClient: HttpClient = defaultHttpClient

export function setHttpClient(client: HttpClient): void {
  httpClient = client
}

export async function dispatch(r2pId: string, endpoint: string, payload: unknown): Promise<DeliveryResult>
```

### Execution steps

1. **Transition to `sent`** — `r2pRepo.update(r2pId, { status: 'sent', updated_at: now }, current.version)`
   - `transitionRepo.append({ r2p_id, from_status: 'created', to_status: 'sent', actor: 'routing-engine' })`

2. **Record dispatch attempt** — `auditRepo.append({ event_type: 'DELIVERY_DISPATCHED', actor: 'routing-engine', detail: JSON.stringify({ endpoint }) })`

3. **HTTP POST** — `httpClient(endpoint, payload)` where payload is:
   ```typescript
   { r2pId, payerId, payeeId, amount, currency, dueDate }
   ```

4. **On HTTP 2xx** — `auditRepo.append({ event_type: 'DELIVERY_CONFIRMED', detail: JSON.stringify({ statusCode }) })`
   - Return `{ status: 'delivered', statusCode }`

5. **On non-2xx or error** — `auditRepo.append({ event_type: 'DELIVERY_FAILED', detail: JSON.stringify({ statusCode, error }) })`
   - Return `{ status: 'failed', statusCode }`
   - Do not throw — retry is handled by ticket 3.2

### Default HTTP client

```typescript
import * as https from 'https'
import * as http from 'http'

function defaultHttpClient(url: string, body: unknown): Promise<{ statusCode: number }> {
  return new Promise((resolve, reject) => {
    const data = JSON.stringify(body)
    const parsed = new URL(url)
    const lib = parsed.protocol === 'https:' ? https : http
    const req = lib.request(
      { hostname: parsed.hostname, port: parsed.port, path: parsed.pathname,
        method: 'POST', headers: { 'Content-Type': 'application/json', 'Content-Length': Buffer.byteLength(data) } },
      (res) => resolve({ statusCode: res.statusCode ?? 0 })
    )
    req.on('error', reject)
    req.write(data)
    req.end()
  })
}
```

## Acceptance Criteria

- [ ] `dispatch()` transitions status `created → sent` before making the HTTP call
- [ ] `DELIVERY_DISPATCHED` audit entry recorded with endpoint
- [ ] HTTP 2xx returns `{ status: 'delivered' }` and records `DELIVERY_CONFIRMED`
- [ ] Non-2xx returns `{ status: 'failed' }` and records `DELIVERY_FAILED` without throwing
- [ ] Network error returns `{ status: 'failed' }` and records `DELIVERY_FAILED` without throwing
- [ ] `createRequest()` calls `dispatch()` fire-and-forget after persisting
- [ ] All unit tests pass with mocked HTTP client (no real HTTP in tests)
