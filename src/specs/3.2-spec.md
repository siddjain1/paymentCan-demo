## Goal

Add exponential backoff retry to `dispatch()` in the Routing Engine. On non-2xx or network error, retry up to 3 times with exponential backoff (max 30s total). After all attempts exhausted, transition status to `delivery_failed` and emit an originator notification stub.

## Retry Design

```
Attempt 1 (initial):  no delay
Attempt 2 (retry 1):  1 000 ms
Attempt 3 (retry 2):  2 000 ms
Attempt 4 (retry 3):  4 000 ms
                      ────────
Total max wait:        7 000 ms  (well under 30 s cap)
```

Constants:
```typescript
const MAX_ATTEMPTS = 4        // 1 initial + 3 retries
const BASE_DELAY_MS = 1_000
```

Backoff formula: `delay = BASE_DELAY_MS * 2 ** retryIndex` where `retryIndex` starts at 0.

## Injectable Sleep

Add alongside the existing `httpClient` injection pattern:

```typescript
type SleepFn = (ms: number) => Promise<void>

let sleepFn: SleepFn = (ms) => new Promise((r) => setTimeout(r, ms))

export function setSleepFn(fn: SleepFn): void { sleepFn = fn }
export function resetSleepFn(): void { sleepFn = (ms) => new Promise((r) => setTimeout(r, ms)) }
```

## Updated dispatch() logic

Replace the single HTTP call with a retry loop:

```typescript
for (let attempt = 0; attempt < MAX_ATTEMPTS; attempt++) {
  if (attempt > 0) {
    await sleepFn(BASE_DELAY_MS * 2 ** (attempt - 1))
  }

  auditRepo.append({ event_type: `DELIVERY_ATTEMPT_${attempt + 1}`, detail: JSON.stringify({ endpoint, attempt: attempt + 1 }) })

  try {
    const response = await httpClient(endpoint, payload)
    if (response.statusCode >= 200 && response.statusCode < 300) {
      auditRepo.append({ event_type: 'DELIVERY_CONFIRMED', detail: JSON.stringify({ statusCode: response.statusCode, attempt: attempt + 1 }) })
      return { status: 'delivered', statusCode: response.statusCode }
    }
    lastError = `HTTP ${response.statusCode}`
    lastStatusCode = response.statusCode
  } catch (err) {
    lastError = err instanceof Error ? err.message : String(err)
  }
}
```

On exhaustion:
1. `r2pRepo.update(r2pId, { status: 'delivery_failed', updated_at: now }, current.version)`
2. `transitionRepo.append({ from_status: 'sent', to_status: 'delivery_failed', actor: 'routing-engine' })`
3. `auditRepo.append({ event_type: 'DELIVERY_EXHAUSTED', detail: JSON.stringify({ attempts: MAX_ATTEMPTS, lastError }) })`
4. `auditRepo.append({ event_type: 'ORIGINATOR_NOTIFIED', detail: JSON.stringify({ reason: 'delivery_failed' }) })` — stub for ticket 6.5
5. Return `{ status: 'failed', error: lastError, statusCode: lastStatusCode }`

Replace existing `DELIVERY_DISPATCHED` audit with `DELIVERY_ATTEMPT_1` on the first attempt — the dispatch is now represented by attempt entries.

## Acceptance Criteria

- [ ] Retries up to 3 times (4 total attempts) on non-2xx or network error
- [ ] Exponential backoff: delays are 1s, 2s, 4s — total ≤ 30s
- [ ] Success on any attempt returns `{ status: 'delivered' }` immediately
- [ ] After 4 failed attempts, status transitions to `delivery_failed`
- [ ] `DELIVERY_EXHAUSTED` audit entry appended after all retries exhausted
- [ ] `ORIGINATOR_NOTIFIED` audit entry appended as notification stub
- [ ] `delivery_failed` state transition recorded in transitionRepo
- [ ] All unit tests pass with injected no-op sleep (no real waits in tests)
