# Spec: 4.1 — POST /r2p/requests/{r2pId}/acknowledge

**Ticket:** 4.1
**Type:** Backend API
**Points:** 3
**Depends on:** 3.1

---

## Endpoint

```
POST /r2p/requests/{r2pId}/acknowledge
```

---

## Input

```typescript
{
  participantId: string   // required — receiving participant ID
  receivedAt:   string   // required — ISO 8601 datetime
}
```

---

## Output

```typescript
{ r2pId: string, status: "delivered" }
```

---

## Errors

| Code | HTTP | Condition |
|------|------|-----------|
| NOT_FOUND | 404 | r2pId does not exist |
| ALREADY_ACKNOWLEDGED | 409 | acknowledgement already recorded for this r2pId |

---

## Logic

### Guards (evaluated in order)

1. **Fetch** — look up request by r2pId. Return 404 NOT_FOUND if missing.
2. **Duplicate check** — if an entry already exists in the R2PAcknowledgement table for this r2pId, return 409 ALREADY_ACKNOWLEDGED.

### Persistence

Persist a `R2PAcknowledgement` row before state mutation:

```typescript
{
  r2p_id:         string   // the r2pId
  participant_id: string   // from input
  received_at:    string   // from input
  created_at:     string   // now
}
```

### State transition

| from | to |
|------|----|
| `sent` (or any current status) | `delivered` |

### Post-transition actions

- Append `REQUEST_ACKNOWLEDGED` audit entry with `actor = participantId`
- Append state transition record with `from_status = current.status`, `to_status = "delivered"`, `actor = participantId`
- Emit `acknowledged` event (stub — ticket 6.5): append `EVENT_ACKNOWLEDGED_EMITTED` audit entry with `actor = "event-publisher"`
