## Errors

| Code | HTTP | Condition |
|------|------|-----------|
| R2P_NOT_FOUND | 404 | r2pId does not exist in R2PRequest store |
| INVALID_STATE_TRANSITION | 409 | request status ‚âÝ `accepted` |
| AMOUNT_MISMATCH | 400 | paymentAmount ‚âÝ request.amount |

Guards evaluated in order: R2P_NOT_FOUND ‚Üí INVALID_STATE_TRANSITION ‚Üí AMOUNT_MISMATCH.

## R2PPayment row

```typescript
{
  payment_id:   string   // randomUUID()
  r2p_id:       string
  amount:       number
  currency:     string
  payer_id:     string
  payee_id:     string
  status:       'processing' | 'settled' | 'failed'
  submitted_at: string   // ISO 8601 now
  settled_at?:  string
}
```

Stored in in-memory Map keyed by `payment_id`. Separate map `paymentsByR2PId` for lookup by r2pId.

---

## Logic

1. **Fetch** r2pId from r2pRepo ‚Äî 404 R2P_NOT_FOUND if missing
2. **State guard** ‚Äî status must be `accepted` ‚Äî 409 INVALID_STATE_TRANSITION
3. **Amount guard** ‚Äî paymentAmount must equal request.amount ‚Äî 400 AMOUNT_MISMATCH
4. **Persist** R2PPayment row with `status: "processing"`
5. **Transition** r2p status: `accepted` ‚Üí `payment_processing`
6. **Audit** `PAYMENT_SUBMITTED` with actor `payment-engine`
7. **State transition** record: `accepted` ‚Üí `payment_processing`
8. **Fire stub** settlement rail (fire-and-forget, deferred to next tick):
   - Default: injectable async fn that resolves after 500ms
   - On resolve: appends `SETTLEMENT_CALLBACK` audit entry (full transition handled in ticket 5.2)
9. **Return** `{ paymentId, r2pId, status: "processing" }`

## Settlement rail stub

```typescript
export type SettlementRailFn = (paymentId: string) => Promise<{ success: boolean }>

// Default: auto-success after 500ms
const defaultRail: SettlementRailFn = (paymentId) =>
  new Promise((resolve) => setTimeout(() => resolve({ success: true }), 500))
```

Injectable via `setSettlementRail(fn)` / `resetSettlementRail()` ‚Äî same pattern as routingEngine.
