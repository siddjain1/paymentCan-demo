# Spec: S1.1 — RtP Creation API Endpoint

**Ticket:** S1.1  
**Requirement:** RTP-INIT-001  
**Branch:** feature/S1.1-rtp-creation-api  
**Epic:** EPIC-01 — Request Initiation and Routing

---

## Endpoint

```
POST /rtp/v1/requests
Content-Type: application/xml
Accept: application/xml
```

mTLS required on all connections. Participant certificate CN must match `OriginatorParticipantId` in the pain.013 message.

---

## Processing Order

1. **Auth** — Validate mTLS cert; look up `OriginatorParticipantId` in participant registry → 403 + `BE01` if missing or inactive
2. **Parse** — Deserialise `pain.013.001.07` XML body
3. **Validate** — Mandatory field check (delegated to S1.2 validator)
4. **Duplicate check** — SHA-256 composite key lookup (delegated to S1.4)
5. **Persist** — Write `RtPTransaction` + `RtPStateTransition(SUBMITTED)` + `RtPMessageStore` in a single DB transaction; generate UUID v4 `RtpTransactionId`
6. **Route** — Call `resolveParticipant(debtorAccount)`; forward pain.013 to payer FI endpoint
7. **State transition** — Write `RtPStateTransition(SUBMITTED → DELIVERED)`; emit `RTP_SUBMITTED` and `RTP_DELIVERED` events
8. **Respond** — Return 202 + `pain.002 ACCP` with `RtpTransactionId`

No routing call is made before steps 1–4 complete successfully. If any step fails, subsequent steps are skipped.

---

## Request

Body: ISO 20022 `pain.013.001.07` XML (Payments Canada variant).

Mandatory fields (validated in step 3):

| Field (pain.013 path) | Reason code if missing |
|---|---|
| `GrpHdr/MsgId` | `MS03` |
| `GrpHdr/CreDtTm` | `MS03` |
| `GrpHdr/NbOfTxs` | `MS03` |
| `PmtInf/CdtTrfTx/Amt/InstdAmt` | `MS03` / `AM03` |
| `PmtInf/CdtTrfTx/Cdtr/Nm` | `MS03` |
| `PmtInf/CdtTrfTx/CdtrAcct/Id` | `MS03` |
| `PmtInf/CdtTrfTx/Dbtr/Nm` | `MS03` |
| `PmtInf/CdtTrfTx/DbtrAcct/Id` | `MS03` |
| `PmtInf/ReqdExctnDt` | `MS03` / `DT01` |

## Responses

### 202 Accepted — ACCP

```xml
<Document>
  <pain.002>
    <GrpHdr>
      <MsgId>{uuid}</MsgId>
      <CreDtTm>{iso8601}</CreDtTm>
    </GrpHdr>
    <OrgnlGrpInfAndSts>
      <OrgnlMsgId>{original MsgId}</OrgnlMsgId>
      <GrpSts>ACCP</GrpSts>
    </OrgnlGrpInfAndSts>
    <OrgnlPmtInfAndSts>
      <TxInfAndSts>
        <OrgnlInstrId>{RtpTransactionId}</OrgnlInstrId>
        <TxSts>ACCP</TxSts>
      </TxInfAndSts>
    </OrgnlPmtInfAndSts>
  </pain.002>
</Document>
```

### 400 Bad Request — RJCT (routing failure)

`pain.002` with `TxSts=RJCT`, `StsRsnInf/Rsn/Cd=AC01`, `AddtlInf="Debtor account not routable"`.

### 403 Forbidden

`pain.002` with `TxSts=RJCT`, `StsRsnInf/Rsn/Cd=BE01`.

### 503 Service Unavailable

Returned when DB write fails after 3 UUID collision retries. No `RtpTransactionId` issued.

## Data Written

### RtPTransaction

| Field | Value |
|---|---|
| `rtpTransactionId` | UUID v4 |
| `originatorParticipantId` | from mTLS cert CN |
| `debtorAccountId` | from `DbtrAcct/Id` |
| `creditorAccountId` | from `CdtrAcct/Id` |
| `instructedAmount` | from `InstdAmt` |
| `currency` | `CAD` |
| `currentState` | `DELIVERED` (on success) |
| `duplicateDetectionKey` | SHA-256(MsgId + CreDate + OriginatorParticipantId) |
| `createdAt` | UTC now |

### RtPStateTransition (two rows)

| row | fromState | toState | triggeredBy |
|---|---|---|---|
| 1 | `null` | `SUBMITTED` | `pain.013 accepted` |
| 2 | `SUBMITTED` | `DELIVERED` | `resolveParticipant success` |

## Constraints

- Fixed amount only — no partial amounts, ranges, or minimum indicators
- All data persisted within Canada
- UUID collision: retry up to 3 times then return 503
- Atomicity: `RtPTransaction` + `RtPStateTransition` + `RtPMessageStore` in one DB transaction

---

## Acceptance Criteria

- [ ] Valid `pain.013` → 202 + unique `RtpTransactionId` in `pain.002 ACCP`
- [ ] Request routed to payer FI in same processing cycle; state reaches `DELIVERED`
- [ ] Unauthenticated or unentitled participant → 403 + `BE01`
