# Spec: S1.2 — Mandatory Field Validation

**Ticket:** S1.2  
**Requirement:** RTP-INIT-002  
**Branch:** feature/S1.2-mandatory-field-validation  
**Epic:** EPIC-01 — Request Initiation and Routing  
**Depends on:** S1.1

---

## Overview

Extracts all pain.013 field validation into a dedicated `pain013Validator.ts` service. The validator operates on the already-parsed `Pain013Fields` struct (produced by `pain013Parser.ts`) and returns a typed result that the route handler converts to a pain.002 RJCT response.

Validation must run **after** parsing, **before** duplicate detection and routing.

---

## Validator Interface

```typescript
// src/services/pain013Validator.ts

export type ValidationFailure = {
  valid: false
  reasonCode: 'MS03' | 'AM03' | 'DT01'
  field: string       // ISO 20022 XPath-style field reference
  detail: string      // Human-readable, goes into AddtlInf
}

export type ValidationResult =
  | { valid: true }
  | ValidationFailure

export function validatePain013Fields(fields: Pain013Fields): ValidationResult
```

## Validation Rules (in evaluation order)

| # | Field | Condition | Reason Code | AddtlInf |
|---|---|---|---|---|
| 1 | `GrpHdr/MsgId` | Present and non-empty | `MS03` | `Missing field: GrpHdr/MsgId` |
| 2 | `GrpHdr/CreDtTm` | Present and non-empty | `MS03` | `Missing field: GrpHdr/CreDtTm` |
| 3 | `GrpHdr/NbOfTxs` | Present and non-empty | `MS03` | `Missing field: GrpHdr/NbOfTxs` |
| 4 | `PmtInf/ReqdExctnDt` | Present and non-empty | `MS03` | `Missing field: PmtInf/ReqdExctnDt` |
| 5 | `PmtInf/ReqdExctnDt` | ≥ today (UTC, YYYY-MM-DD) | `DT01` | `RequestedExecutionDate must not be in the past` |
| 6 | `PmtInf/CdtTrfTx/Amt/InstdAmt/@Ccy` | Equals `"CAD"` | `AM03` | `Currency must be CAD` |
| 7 | `PmtInf/CdtTrfTx/Amt/InstdAmt` | > 0 | `AM03` | `InstructedAmount must be greater than 0` |
| 8 | `PmtInf/Cdtr/Nm` | Present and non-empty | `MS03` | `Missing field: PmtInf/Cdtr/Nm` |
| 9 | `PmtInf/CdtrAcct/Id` | Present and non-empty | `MS03` | `Missing field: PmtInf/CdtrAcct/Id` |
| 10 | `PmtInf/Dbtr/Nm` | Present and non-empty | `MS03` | `Missing field: PmtInf/Dbtr/Nm` |
| 11 | `PmtInf/DbtrAcct/Id` | Present and non-empty | `MS03` | `Missing field: PmtInf/DbtrAcct/Id` |
| 12 | `XpryDt` (optional) | If present: must be > `GrpHdr/CreDtTm` | `DT01` | `ExpiryDateTime must be after CreationDateTime` |

Fields 1–4 and 8–11 are presence checks — already enforced by the parser returning `{ ok: false, missingField }`. The validator re-enforces them on the parsed struct as a belt-and-suspenders check and to centralise all reason-code mapping in one place.

Fail fast: return on the **first** failure. Do not accumulate multiple errors.

---

## Processing Order in Route Handler

```
1. Auth (BE01)
2. Parse XML → Pain013Fields   [parser — MS03 on missing field]
3. Validate fields             [validator — MS03 / AM03 / DT01]
4. Duplicate detection         [ED05]
5. Resolve routing             [AC01]
6. Persist + state transition
7. pain.002 ACCP response
```

---

## Route Handler Changes

Remove inline validation from `src/routes/rtpRequests.ts` (currency, amount, date checks) and replace with a single call:

```typescript
const validation = validatePain013Fields(f)
if (!validation.valid) {
  xmlReply(res, 400, buildRjctResponse(f.msgId, validation.reasonCode, validation.detail))
  return
}
```

---

## pain.002 RJCT Structure

```xml
<TxInfAndSts>
  <TxSts>RJCT</TxSts>
  <StsRsnInf>
    <Rsn>
      <Cd>{reasonCode}</Cd>       <!-- MS03 | AM03 | DT01 -->
    </Rsn>
    <AddtlInf>{detail}</AddtlInf> <!-- failing field or description -->
  </StsRsnInf>
</TxInfAndSts>
```

---

## Acceptance Criteria

- [ ] Missing any of the 9 mandatory fields → 400 + `MS03`, `AddtlInf` identifies the field
- [ ] `ReqdExctnDt` in the past → 400 + `DT01`
- [ ] Currency ≠ `CAD` → 400 + `AM03`
- [ ] `InstdAmt` = 0 → 400 + `AM03`
- [ ] All fields valid → validation passes, processing continues to duplicate check
- [ ] No routing call made before validation passes
- [ ] `ExpiryDateTime` present and ≤ `CreDtTm` → 400 + `DT01`
